# Cortex - Workflows

1. **New Feature Development:**
   - Update `PLAN.md` with the feature details.
   - Implement the feature using the approved tech stack.
   - Add the feature to the `[Unreleased]` section of `CHANGELOG.md`.

2. **Releasing a Version:**
   - Change `[Unreleased]` to the new semantic version in `CHANGELOG.md`.
   - Update version links at the bottom of the changelog.
   - Run `git commit -m "feat: release version X.Y.Z"`.
